Java集合框架

发表于 2018-10-20 | 分类于 Java | 阅读次数：

Collection

  • AbstractCollection是其父级抽象类
  • 是Set，Queue，List的父接口

Set

  • 父级接口为Collection，拥有此接口中定义的所有方法
  • AbstractSet为其父级抽象类，扩展自AbstractCollection抽象类
  • 无序、不可重复的集合
  • 无额外方法，但不允许添加重复元素，add() 方法在添加重复元素时会返回false
HashSet
  • 无序，唯一
  • 基于HashMap实现，底层采用HashMap保存元素
  • 线程不安全
  • 值可以为Null
  • 更适合添加、查询操作
LinkedHashSet
  • 有序，唯一
  • 继承自HashSet，元素会按照插入顺序进行排序
  • 底层通过LinkedHashMap实现
  • 更适合插入排序、删除以及遍历操作
TreeSet
  • 有序，唯一
  • 父级接口为SortedSet
  • 红黑树，自平衡的排序二叉树
  • 在实例化时可通过一个自定义比较器Comparator来指定元素顺序
  • 额外方法：
    • lower()，返回小于给定元素的元素
    • floor()，返回小于等于给定元素的元素
    • ceiling()，返回大于等于给定元素的元素
    • higher()，返回大于给定元素的元素
    • 如果不存在这样的元素，则返回 null。
EnumSet

  • 与枚举类型一起使用的专用Set实现

  • 所有元素必须来自单个枚举类型

  • 无构造方法，只能通过内置静态方法从指定的枚举类中获取实例

  • 如：EnumSet<EnumColor> allSet = EnumSet.allOf(EnumColor.class);

  • 方法

    • allOf()
    • clone()
    • noneOf()

List

  • 父级接口为Collection，
  • AbstractList为其父级抽象类，扩展自AbstractCollection抽象类
  • 有序、可以重复的集合
  • 每个元素都有其对应的顺序索引，允许在指定位置进行操作，且增加了迭代器ListIterator
  • 父级接口为Collection，拥有此接口中定义的所有方法

ArrayList ⭐️⭐⭐️
  • 数组队列，相当于一个动态数组，由数组实现
  • 随机访问效率高，但随机插入、删除效率低
  • 线程不安全，但效率相对而言较高
Vector
  • 矢量队列，相当于一个动态数组，由数组实现
  • 线程安全，但效率相对而言较低

######Stack

  • 栈，继承与Vector
  • 先进后出(FILO, First In Last Out)
  • 应优先使用LinkedList替代

LinkedList ⭐️⭐⭐️
  • 双向链表，也可以被当作一个栈、队列或者双向队列进行操作
  • 随机访问效率低，随机插入、删除效率高
  • 线程不安全

Queue

具有先进先出（FIFO）队列关系的集合

BlockingQueue

此为Queue的子级接口

阻塞队列

当队列满了的时候进行入队操作，会被阻塞，除非另一个线程进行出队操作

当队列空了的时候进行出队操作时，会被阻塞，除非另一个线程进行入队操作

线程安全

不能插入Null值，否则会抛出NullPointerException

######LinkedBlockingQueue

  • 可通过传值设置队列大小，默认为Integer最大值
  • 内部通过链表实现
ArrayBlockingQueue
  • 创建时bible设置队列大小
  • 内部通过数组实现
PriorityBlockingQueue
  • 队列默认初始大小为8，最大值为Integer最大值-8
  • 内部通过数组实现
  • 可以插入Null
  • 所有插入元素必须实现Comparable接口，队列优先顺序按照此比较器实现

SynchronousQueue

  • 无数据缓冲
  • 生产者的put必须等待消费者的take，反之亦然
  • 创建时设置为公平竞争或非公平竞争，默认为非公平竞争
  • 非公平使用先进后出
  • 公平竞争使用先进先出
  • 通过链表实现

######DelayQueue ⭐️⭐⭐️

  • 阻塞，有序，延迟
  • 所有插入元素必须实现java.util.concurrent.Delayed接口，用于设置元素延迟释放时间
  • 通过getDelay()获取延迟释放时间，返回0或者负值，则表示元素到了释放时间，会通过take方法释放该元素
  • 应用场景很多：
    • 关闭空闲连接
    • 缓存
    • 任务超时处理

#####Deque

此为Queue的子级接口

双端队列，可以从队列的两端进行插入、弹出和删除操作

LinkedList
  • 同时实现了List和Deque接口，既可以看作一个顺序容器，也可以看作是一个队列，还可以看作一个栈
  • 内部通过双向链表实现
ArrayDeque
  • 内部通过数组实现
  • 创建时指定队列大小，默认为16，最小为8
  • 比LinkedList效率高

Map

具有映射关系的集合

key不允许重复，所有的key放在一起，可以看成一个Set集合

value放在一起，类似一个List

HashMap
  • 线程不安全，但性能相对而言较高
  • key与value允许使用Null
HashTable
  • 线程安全，但性能相对而言较低
  • key与value都不允许为Null
ConcurrentHashMap ⭐️⭐⭐️
  • 线程安全，且性能比HashTable高
  • 父级接口为ConcurrentMap
LinkedHashMap
  • 继承自HashMap
  • 有序，通过使用双向链表来维护key-value对的次序
  • 插入性能较低，但是在迭代访问时时性能较好

######TreeMap

  • 继承自SortedMap接口
  • 有序，通过红黑树实现
  • 自然排序，key必须实现Comparable接口，且所有key应该时同一个类的对象，否则抛出ClassCastException
  • 定制排序，需在创建时传入Comparator对象，用于对所有key进行排序
