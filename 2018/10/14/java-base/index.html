Java基础

发表于 2018-10-14 | 分类于 Java | 阅读次数：

Java基础知识点总结，仅供复习时使用

数据类型
  • 基本数据类型
    • byte
      • 8位、有符号 整数
      • 最小值是-128(-2^7)
      • 最大值是127(2^7-1)
      • 默认值是0
      • 占用空间是int的四分之一
      • 包装类 Byte
    • short
      • 16位、有符号 整数
      • 最小值是-32768(-2^15)
      • 最大值是32767(2^15 - 1)
      • 默认值是0
      • 占用空间是int的二分之一
      • 包装类 Short
    • int
      • 整数默认类型
      • 32位、有符号 整数
      • 最小值是-2,147,483,648(-2^31)
      • 最大值是2,147,485,647(2^31 - 1)
      • 默认值是0
      • 包装类 Integer
    • long
      • 64位、有符号整数
      • 最小值是-9,223,372,036,854,775,808(-2^63)
      • 最大值是9,223,372,036,854,775,807(2^63 -1)
      • 默认值是0L
      • 包装类 Long
    • float
      • 单精度、32位的浮点数
      • 默认值是0.0F
      • 不能用于表示精确的值
      • 包装类 Float
    • double
      • 浮点数默认类型
      • 双精度、64位的浮点数
      • 默认值是0.0D
      • 不能用于表示精确的值
      • 包装类 Double
    • boolean
      • true 或者 false
      • 默认false
      • 包装类 Boolean
    • char
      • 单一的16位Unicode字符
      • 最小值是'\u0000'（即为0）
      • 最大值是'\uffff'（即为65,535）
      • 默认值是'\u0000'（即为0）
      • char数据类型可以储存任何字符
      • 使用单引号声明
      • 包装类 Character
  • 引用数据类型
    • Array
      • 声明
        • dataType[] arrayRefVar;
        • dataType[] arrayRefVar = new dataType[arraySize];
        • dataType[] arrayRefVar = {value0, value1, ..., valuek};
      • 内置方法
        • .length
    • Class
    • Interface
  • 常量
    • 使用final关键字声明
    • 一般使用大写作为标识符
  • 类型转换
    • 规则
      • boolean不能转换
      • 转换类型应该是相关类的对象
      • 将大容量类型转小容量类型，需强制转换
      • 转换过程可能会导致溢出或损失精度
      • 浮点转整数，会舍弃小数，而不是四舍五入
    • 自动类型转换
      • 转换钱的数据类型位数要低于转换后的数据类型
      • byte,short,char-> int -> long -> float -> double
    • 强制类型转换
      • 转换的数据类型必须兼容
      • 格式为：(type) value
    • 隐含强制类型转换
      • 整数默认为int
      • 浮点不存在此情况，在定义时必须在数字后跟上F或f
变量
  • 声明
    • 变量在使用前必须声明
    • 声明格式为type identifier [ = value][, identifier [= value] ...] ;
  • 局部变量
    • 声明在方法、构造方法、或语句块中
    • 局部变量在执行完成后，将会被销毁
    • 访问修饰符不能用于局部变量
    • 只在声明局部可见
    • 局部变量时在栈上分配的
    • 局部变量没有默认值，必须初始化！
  • 实例变量
    • 声明在类中，但在方法、构造方法、语句块之外
    • 随着对象被实例化而确定
    • 在对象被创建时创建、对象被销毁时销毁
    • 应保证被使用
    • 可以声明在使用前或使用后
    • 可以被访问修饰符修饰
    • 具有默认值，可以在声明时指定，或在构造方法中指定
    • 在静态方法或其他类中，应使用完全限定名访问
  • 静态变量
    • 类中以static声明， 但必须在方法、构造方法、语句块之外
    • 无论创建了多少对象，类值拥有静态变量的拷贝
    • 一般被声明为常量
    • 在第一次被访问时创建，程序结束时销毁
    • 可以被访问修饰符修饰，一般被声明为public
    • 具有默认值，和实力变量相似
    • 通过完全限定名访问

字符串

  • 声明

    • 使用双引号括起来"
    • 字符串都是String类的一个实例
    • 使用加号+进行字符串拼接
    • 字符串不可变

  • 内置方法

    • .contains(str)

      判断是否包含某个子串

    • .equals(str)

      检测字符串是否相等

    • .equalsIgnoreCase(str)

      检测字符串是否相等，且不区分大小写

    • .indexOf(str) /.lastIndexOf(str)

      查找字符串并返回匹配的第一个子串位置，这个位置从串头/串尾开始计算

    • .join(delimiter, ...ele)

      将所有元素使用定界符进行拼接，返回一个字符串

    • .split(delimiter)

      将字符串根据给定的定界符进行分割，返回一个数组

    • .length()

      获取字符串长度

    • .replace(oldStr, newStr)

      将字符串中的某段子串使用新的子串替换

    • .substring(start, end)

      截取字符串，获取一个子串

    • .startWith(str) / .endWith(str)

      判断字符串是否以给定字符为开头或结尾

    • .toLowerCase() /.toUpperCase()

      将字符串转为小写/大写

    • .trim()

      删除首尾空格

  • 构建字符串

    • StringBuilder
      • 线程不安全，但是效率高
      • 在非循环的情况下，使用+进行字符串拼接会隐性地调用此方法
    • StringBuffer
      • 线程安全，但是效率较低
      • 正常情况下不会考虑使用此对象进行字符串构建，即便有此需求，也会考虑使用锁
    • 内置方法
      • .append(str) 追加一个字符串
      • .insert(offset, str) 在指定位置插入一个字符串
      • .length() 获取当前构建字符串的长度
      • .delete(start, end) 删除置顶区间的字符
      • .setCharAt(offset, char) 修改指定位置的字符
      • .toString() 构建完毕，返回一个字符串对象

  • 注意

    • 在调用内置方法之前，应当首先判断不为null，否则将会抛出异常

  • 内置方法

    • Arrays.copyOf(array, array.length()) 拷贝数组，直接通过等号赋值，实际是将引用复制了过去

异常

  • 异常类基础结构

  • 基础使用方式

  • 手动抛出异常

    • throws
      • 通过使用throws关键字声明方法抛出的异常
      • throws 关键字放在方法签名的尾部
      • 一个方法可以声明抛出多个异常
    • throw
      • 代码块中通过使用 throw 关键字强制抛出一个异常

  • 声明自定义异常

    • 所有异常都必须是 Throwable 的子类
    • 检查性异常类，需要继承 Exception 类
    • 运行时异常类，需要继承 RuntimeException 类

  • 注意事项

    • catch 不能独立于 try 存在
    • 在 try/catch 后面添加 finally 块并非强制性要求
    • try 代码后不能既没 catch 块也没 finally 块
    • try, catch, finally 块之间不能添加任何代码
